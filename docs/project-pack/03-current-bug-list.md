# 当前已知 Bug 清单

这份清单按“适合继续进入下一轮排查 / 回归”的视角整理，既包含未修问题，也包含已修但需要重点回归验证的问题。

## P0

### 1. 周排程到周结算的状态链路历史上非常脆弱

- 现象：
  - 玩家逐天安排了一周行动，但结算时曾多次出现“表面选了，实际全按摆烂处理”的问题。
- 可能原因：
  - 前端状态、确认 payload、结算器读取源不一致
  - 默认摆烂补全覆盖真实选择
  - 事件 / 行动池回退时导致真实行动丢失
- 当前状态：
  - 已修待高强度回归
  - 最近相关修复集中在 `d23a707`、`03db403`、`b148926`

### 2. 周排程相关规则回归风险高

- 现象：
  - 当前周排程涉及课程态度、事件落位、逐天选择、默认补全、结算读取、周前现金预警，链路长且耦合点多。
- 可能原因：
  - 多模块共同作用：`app/game`、`components/action-plan-form`、`core/game-engine/monthly.ts`、`core/resolvers/schedule.ts`
- 当前状态：
  - 已修待验证
  - 不建议在未做回归测试前继续大改这条链路

## P1

### 1. 后半程路径内容深度不均衡

- 现象：
  - 路径“形成感”已经有了，但不同路线后续内容的厚度不一致，玩家可能会感觉有些方向更像真实路径，有些方向更像框架。
- 可能原因：
  - 规则层先接通了骨架，但内容池和事件池扩充还没跟上
- 当前状态：
  - 未修
  - 更适合列入后续内容轮，而不是当成当前致命 bug

### 2. 竞赛 / 长期项目仍偏首发骨架

- 现象：
  - 已经有入口、关闭、投入和奖项，但“过程感”和阶段差异仍偏弱。
- 可能原因：
  - 当前实现优先把规则链路跑通，没有做厚内容
- 当前状态：
  - 未修

### 3. 前台解释层仍需持续回归

- 现象：
  - 成长日志、月记、履历已经开始解释“为什么会走向某条路”，但每轮规则改动后都可能出现前台提示和规则层不完全同步的问题。
- 可能原因：
  - 方向提示、奖学金说明、推免说明、公考说明都依赖多处汇总 helper
- 当前状态：
  - 已修待持续验证

- 可能原因：

### 2. 结局内容库仍偏框架化

- 现象：
  - 结局结构已通，但不同结局的文本厚度和分支细节还不够
- 可能原因：
  - 当前优先级一直放在周排程、成长路径和前台可感知化
- 当前状态：
  - 未修

## 当前建议

如果要继续开新一轮开发，最值得优先盯住的不是“再加系统”，而是：

1. 周排程到周结算链路回归
2. 比赛 / 长期项目线回归
3. 方向形成解释是否和规则层同步
4. 月末成长日志 / 月记 / 结局预览是否还能稳定读取摘要
