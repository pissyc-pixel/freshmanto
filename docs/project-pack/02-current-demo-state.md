# 当前 Demo 实现状态

本文对应的是当前仓库代码状态，而不是完整 PRD 理想形态。

## 总体判断

当前 Demo 已经有一个明确的可玩闭环：

- 开局生成角色
- 进入按周排程
- 每周先定课程态度
- 周前随机事件落到具体某天
- 逐天安排一周
- 统一进行周结算
- 月末产生成长日志、月记、履历与结局预览

它已经不是纯概念，但距离“完整四年人生模拟”还有明显距离。

## 1. 开局与基础画像

### 已实现

- 随机开局
- 学校档次 / 城市档次等基础画像
- 初始统计值生成

### 半实现

- 学院 / 专业风味已开始影响后续系统，但内容密度仍有限

### 未实现

- 更细颗粒度的个性特质系统
- 更丰富的开局家庭与人格分化

## 2. 周排程与时间系统

### 已实现

- 当前主流是按周推进
- 周开始先定课程态度
- 周前事件落位到具体天
- 每天 1 个行动
- 全周确认后统一结算
- 未选日期自动补默认摆烂
- 周结算支持逐日结果与本周总变化

### 半实现

- 时间规则已经按“白天满课 / 半天 / 全天可支配”运行
- 但更细的时间块、跨天连续投入、复杂冲突处理还不够丰富

### 未实现

- 真实课表粒度
- 节次级别冲突
- 更复杂的周内多时段资源管理

## 3. 课程与学业

### 已实现

- 课程态度影响周内背景状态
- 翘课可以释放白天时间，并带来学业 / 压力代价
- 学业值会进入 GPA / 排名 / 百分比等推导

### 半实现

- 学业反馈已经接到奖学金、推免、方向形成
- 但课程内容本身仍偏抽象

### 未实现

- 具体课程分科
- 挂科救火链
- 老师风格差异

## 4. 随机事件

### 已实现

- 周开始前触发事件
- 事件落到具体某一天
- ABDE 四类已经开始接入
- 事件会影响当日行动池或产生特殊入口

### 半实现

- 事件结构正确，但事件池内容仍偏少
- 不同学院 / 学校 / 城市的风味差异还不够密

### 未实现

- 大量中后期专属事件
- 更复杂的连锁事件和跨周后果

## 5. 人生导向倾向系统

### 已实现

- 就业 / 考研 / 考公 / 推免 / 未定 等方向倾向状态
- 前台已有“方向正在形成”的轻量解释
- 成长日志、月记、履历页面会读取方向信号

### 半实现

- 趋势表达已经可见
- 但不同路径的后期内容密度仍不均衡

### 未实现

- 更细的方向竞争与切换机制
- 路线之间的长期代价对冲

## 6. 竞赛系统

### 已实现

- 竞赛按长期项目线接入
- 存在入口事件、状态、投入与结算
- 已接触且未关闭的项目线才可继续投入
- 翘掉入口后，本学期该线可关闭
- 奖项会回流到履历和路径画像

### 半实现

- 规则骨架已经到位
- 但竞赛种类、阶段表现、项目差异仍偏首发版

### 未实现

- 更丰富的多阶段竞赛过程
- 团队协作与老师资源影响
- 作品 / 成果中间态展示

## 7. 奖学金系统

### 已实现

- 每学年初结算
- 大一没有奖学金
- 从大二开始按上一学年表现评定
- 结果会进入金钱、履历和说明层

### 半实现

- 已有前台解释“为什么拿到 / 没拿到”
- 但奖学金层级与学校差异还不算丰富

### 未实现

- 更细的奖项等级
- 院系内部名额竞争感

## 8. 推免 / 考研 / 考公 / 就业路径

### 已实现

- 推免资格会在大三下做规则判断
- 公考进度已存在前台解释
- 履历与方向系统已经能投射到后期路径
- 结局层已有结构化预留

### 半实现

- 路径“形成感”已经建立
- 但完整的后期过程内容还不足

### 未实现

- 完整投递 / 面试 / offer 链
- 考研上岸过程细化
- 公考过程细化
- 多种保研 / 直博后续展开

## 9. 成长日志 / 月记 / 履历

### 已实现

- 前台日志已切成“成长日志”
- 月记与成长日志分工明确
- 履历优先展示 GPA、排名、百分比、比赛、实习、奖学金
- 奖学金、推免、公考等已有解释层

### 半实现

- 表达层已经比早期更自然
- 但真实内容密度仍依赖后续系统扩充

### 未实现

- 更长期的角色自我叙事
- 更丰富的人际、情绪和人生感变化

## 10. 结局系统

### 已实现

- 已有结局页与基础结局结构
- AI 结局表达边界明确

### 半实现

- 结构接口已通
- 但结局文本和路线深度仍属于框架版

### 未实现

- 完整四年终局内容库
- 更丰富的成功 / 普通 / 失败 / 转向分层

## 11. 工程与部署

### 已实现

- 本地可运行
- 测试、lint、tsc 已接入
- 有健康检查接口
- 有 VPS 部署文档

### 半实现

- 更像“支持小规模上线”的原型工程
- 尚未形成正式运维体系

### 未实现

- 正式生产监控
- 多环境配置体系
- 账号系统 / 权限模型

## 当前 Demo 与完整 PRD 的主要差距

最大的差距不是“完全没系统”，而是“系统骨架已经有了，但内容密度、稳定性和后期深度还不够”。

主要差距包括：

- 四年长线内容仍偏稀疏
- 后期路径虽然已经形成，但过程内容不够厚
- 事件池、竞赛池、机会池还需要扩量
- 经济、生存压力、履历成长还可以更真实
- 结局层仍偏框架
- 前台解释已经有明显进步，但仍需持续防止“后台很复杂，前台没感觉”
