# freshmanto 项目上下文速览

## 这是什么项目

freshmanto 是一个大学生活模拟 Demo。

它的核心不是“校园装饰皮”，而是想让玩家感受到：

大学四年里，每周怎么排时间、怎么处理学业、金钱、履历、比赛和现实事件，会慢慢把人推向不同未来。

## 当前阶段

当前是“可玩原型 / 内测原型”：

- 已经有可玩闭环
- 规则层已经开始复杂化
- 前台解释层也已经接上
- 但内容密度、稳定性和长期路径深度仍在持续打磨

## 当前最重要的规则边界

1. 玩法主干是按周推进，不是整月一次性抽象结算  
2. 每周先定课程态度，再逐天选行动  
3. 每天只能选 1 个行动  
4. 周前事件会落到具体某天并影响行动池  
5. 全周排完后统一周结算  
6. 月末再汇总成长日志、月记、履历和结局预览  
7. 规则必须代码驱动，不能塞进 AI prompt

## 当前最重要的已接入系统

- 周历排程与周结算
- 成长日志 / 月记 / 履历
- 人生导向倾向
- 竞赛长期项目
- 奖学金
- 推免资格判断
- 公考进度
- ABDE 周前事件
- 结局前置结构

## 当前最重要的未完成内容

- 四年长线内容密度仍不足
- 后期路径已经成形，但过程内容还不够厚
- 竞赛、就业、考研、考公、推免等路线还需要更完整的中后期过程
- 结局仍偏框架

## 当前最值得注意的风险 / bug

- 周排程到周结算这条链路历史上很脆弱，虽然近几轮已修，但必须持续回归验证
- 比赛 / 长期项目线刚建立不久，容易在入口状态、关闭状态、后续投入上出回归问题
- 方向形成提示、奖学金说明、推免说明、公考说明等前台解释层需要持续和规则层保持同步

## AI 在这个项目里绝不能越界做什么

- 不能判规则
- 不能决定玩家成功或失败
- 不能决定事件是否发生
- 不能决定奖学金 / 推免 / 就业结果
- 不能把未实现系统写成“已经有”
- 不能绕过规则层直接编造关键事实

AI 只负责两类表达：

- 月记
- 结局表达

而且它只能基于规则层已经生成的结构化摘要来写。
