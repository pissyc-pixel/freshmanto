# freshmanto 完整 PRD 整理版

## 1. 产品定位

freshmanto 想做的，不是一个“随便点几下就毕业”的校园主题小游戏，而是一个能让玩家切身感受到：

“大学前几年怎么过，会慢慢决定后几年能走哪条路、这条路有多稳、要付出什么代价。”

核心体验目标：

- 不是爽文式一路升级
- 不是毕业前突然弹出职业选择器
- 而是通过时间安排、课程态度、资源约束、履历积累、现实事件和长期方向形成，逐步把玩家推向不同未来

目标情绪：

- 有代入感
- 有现实摩擦感
- 有节奏感
- 有“原来我是这样一步步走到这里”的解释感

## 2. 开局属性

开局应由规则层生成一组基础画像，影响四年体验。

至少包括：

- 学校档次
- 学院 / 专业倾向
- 城市环境
- 家境 / 生活费水平
- 初始学业基础
- 初始社交基础
- 初始情绪 / 抗压水平
- 可能的个体特质或随机底色

设计原则：

- 开局差异要真实影响资源与机会
- 但不能在一开始就把路线完全锁死
- 开局只提供“起点差异”，不直接决定最终命运

## 3. 学院 / 学校 / 城市系统

### 学校档次

学校档次影响：

- 推免竞争基础
- 资源密度
- 讲座 / 项目 / 比赛 / 实习信息机会
- 奖学金与履历含金量的修正
- 就业与深造路径的起点

### 学院 / 专业

学院要影响：

- 课程负担
- 行动池风味
- 比赛风味
- 随机事件类型
- 后期路径的自然倾向

例如：

- 工：电赛、工程设计、实验、项目实践
- 商：案例赛、商赛、宣讲、实习导向更强
- 理：科研小项目、建模、学术积累
- 文：写作、调研、演讲、组织工作
- 医：见习、实践、实验、调研

### 城市

城市要影响：

- 生活成本
- 实习机会密度
- 招聘会 / 宣讲会密度
- 外部资源接入难度
- 日常经济压力

## 4. 时间系统

### 当前目标结构

时间系统的主表达应以“周”为单位，而不是整月一把梭。

当前认可的结构：

1. 周开始先定课程态度
2. 周前随机事件落到具体某天
3. 玩家逐天安排这一周
4. 每天只能安排 1 个行动
5. 一周排完后统一确认
6. 然后统一结算

### 每周时间规则

- 周一 / 周三 / 周五：默认白天满课，不翘课时只允许夜间行动
- 周二 / 周四：默认半天可用，可在半天行动与夜间轻量行动中择一
- 周末：全天可支配，可从全天 / 半天 / 夜间轻量池中择一

### 默认补全

如果玩家漏选某天，系统自动补成“摆烂 / 发呆 / 混过去”：

- 扣固定生活费
- 小幅回心情
- 小幅降压力
- 不推进学业和履历

## 5. 课程与学业系统

### 课程态度

课程态度是每周背景状态，不是即时行动。

它应在一周开始前确定，并影响：

- 这一周课程占用感
- 学业推进速度
- 压力与心情变化
- 后续推免 / 考研基础

### 翘课规则

翘课不再是课程态度的一部分，而是会影响某一天时间可用性的特殊前置条件：

- 主要价值：释放白天时间
- 主要代价：扣学业、加压力
- 不再依赖“找人代课 / 代签到”

### 学业反馈

学业不只影响一个抽象分数，还应逐渐投射为：

- GPA
- 排名
- 百分比
- 推免基础
- 奖学金竞争力
- 考研准备效率

## 6. 金钱系统

金钱系统必须有现实感，不只是奖励值。

### 基础结构

- 家里给的生活费
- 每周 / 每月固定生活成本
- 行动的直接收支
- 特殊收入：兼职、奖学金、家庭额外支持
- 特殊支出：娱乐、大餐、活动、代价事件

### 设计原则

- 普通学生的大多数生活费应接近现实
- 不允许“明明一直在花钱，结果结算时莫名净赚”且无法解释
- 玩家应该能看懂钱为什么变化

### 风险层

当现金过低时，系统要能提前预警：

- 下周固定开销是否扛不住
- 如果不赚钱或不找家里要钱，是否会恶化

## 7. 社交系统

社交不应只是一项装饰数值。

至少应逐步影响：

- 心情和压力
- 信息获取
- 帮助与补救机会
- 履历入口
- 后期就业与资源流向

后续可扩展：

- 朋友关系
- 组织关系
- 恋爱线
- 人脉型帮助

## 8. 履历系统

履历页不只是字段列表，而应成为“你为什么更像走向某条路”的证据板。

优先展示：

- GPA
- 排名
- 百分比
- 比赛
- 实习
- 奖学金

后续可逐步承接：

- 项目
- 志愿活动
- 组织经历
- 老师项目
- 论文 / 科研

## 9. 随机事件系统

当前认可的随机事件大类是 ABDE：

### A. 强制占用类

例如：

- 班会
- 导员通知
- 材料补交
- 签到严查
- 强制讲座

作用：

- 改写当天时间结构
- 压缩行动池
- 增加现实节奏感

### B. 信息机会类

例如：

- 企业宣讲
- 保研说明会
- 考研讲座
- 公考讲座
- 经验分享

作用：

- 给某条未来路径提供可见入口
- 提升信息获取与后续概率

### D. 履历机会类

例如：

- 大创项目申报
- 比赛邀约
- 项目招募
- 实习内推

作用：

- 直接打开履历增长线
- 与比赛、实习、长期方向联动

### E. 学院 / 学校风味类

例如：

- 工科实验周
- 商学院案例赛
- 文科学院调研活动
- 理科学院竞赛集训

作用：

- 强化出身差异
- 让学院 / 学校 / 城市不是背景板

### 事件时机

事件必须在周开始前触发并落到具体某天，对本周实际排程产生影响，而不是纯文案提示。

## 10. 人生导向倾向系统

这是后半程路径形成的核心中层系统。

至少包括：

- 就业倾向
- 考研倾向
- 考公倾向
- 推免倾向
- 未定倾向

设计原则：

- 倾向不是一次性锁死
- 可以并存，但会逐渐形成主导方向
- 必须由规则驱动累积
- 不允许只靠 AI 文案“判断你适合什么”

它应影响：

- 后续行动可见性
- 事件出现权重
- 某些分支成功率
- 月记与成长日志的表达提示

## 11. 竞赛系统

竞赛不是一道题，而是一条长期项目线。

### 竞赛形态

更接近：

- 大创
- 电赛
- 项目赛
- 案例赛
- 建模
- 调研项目

### 核心规则

- 每学期刷新有限的竞赛 / 项目入口
- 入口以说明会 / 招募会 / 报名信息等形式出现
- 只有接触入口且未关闭，后续投入行动才可见
- 明确翘掉入口后，本学期该项目线关闭

### 投入与结算

- 需要累计投入
- 存在保底投入门槛
- 不足门槛：无法形成成果
- 达到门槛后：有概率拿到不同层级奖项
- 投入越高：奖项上限与概率越高

### 奖项层级

- 校级
- 省级 / 区级
- 国家级

每层再分：

- 一等奖
- 二等奖
- 三等奖

### 竞赛影响

- 履历
- 成就感
- 奖学金
- 推免画像
- 老师 / 项目机会
- 就业竞争力

## 12. 奖学金系统

### 时间规则

- 每学年初结算一次
- 大一没有奖学金
- 从大二学年初开始，根据上一学年表现结算

### 判定因素

- GPA
- 排名
- 百分比
- 是否挂科
- 竞赛成绩
- 履历表现
- 学校档次修正

### 结果层

至少支持：

- 无奖学金
- 普通奖学金
- 高等级奖学金

### 奖学金影响

- 金钱
- 成就感
- 履历
- 推免倾向 / 推免资格
- 成长日志与月记内容

## 13. 后期路径：推免 / 考研 / 考公 / 就业

### 共同原则

这些路径不能在毕业前突然弹出，而要随着大学过程逐步形成。

### 推免 / 保研 / 直博

- 大三下进行资格判断
- 依据 GPA、排名、百分比、竞赛、奖学金、学校档次、学院背景等综合判定
- 结果分为：
  - 明确有资格
  - 边缘可争取
  - 基本无资格

即使有资格，也应允许：

- 接受推免
- 放弃推免转考研
- 放弃推免转就业

### 考研

- 大三开始明显出现准备行为
- 大三下与大四上强化
- 前期学业积累影响准备效率与上岸概率

### 考公

- 大三下到大四逐步开放
- 前台展示“公考进度”
- 后台可拆为行测 / 申论等准备度
- 公考进度应影响后续事件、反馈和结果

### 就业

- 从履历、实习、宣讲、投递、面试逐步展开
- 不再是毕业前突然点一下“找工作”
- 学校、学院、城市、履历、社交、实习都要影响求职质量

## 14. 结局结构

### 第一层：毕业状态

- 正常毕业
- 延毕
- 肄业

### 第二层：人生去向主分支

- 就业
- 推免 / 保研 / 直博
- 考研
- 考公

### 第三层：结果层

每条路线至少支持：

- 成功
- 普通
- 失败
- 转向

设计原则：

- 结局先由规则层判定结构
- AI 只把结构结果写成更像人的结局表达

## 15. AI 的表达边界

AI 在项目中的定位必须长期保持克制：

- 只能吃规则层给的结构化摘要
- 只能生成月记和结局表达
- 不能判规则
- 不能编事实
- 不能决定成功与失败
- 不能把 PRD 中未实现的系统“写出来当成已经有了”

这条边界是整个项目最重要的工程约束之一。
