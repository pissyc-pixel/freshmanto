# freshmanto 项目总览

## 项目名称

freshmanto

## 一句话定位

一个以“大学四年如何被时间、资源、压力与选择慢慢推向不同未来”为核心体验的可玩型大学生活模拟 Demo。

## 当前阶段判断

当前更接近“可玩原型 / 内测原型”：

- 已经具备从开局、按周排程、周结算、月结算、成长日志、月记到结局预览的基本闭环。
- 规则层已经不是纯概念稿，很多核心系统已经代码化。
- 但内容密度、系统平衡、长期路径完整度和前台稳定性仍不足以当成正式上线产品。

## 技术栈简述

- 前端：Next.js 16、React 19、TypeScript、Tailwind CSS 4
- 后端集成：Next.js Server Actions / Route Handlers
- 数据层：Supabase、PostgreSQL
- AI 调用：OpenAI SDK
- 校验与类型：Zod、TypeScript
- 测试：Vitest
- 工程质量：ESLint

## 当前部署情况

仓库内已经准备了本地运行与 VPS 部署方案：

- 本地开发：`npm run dev`
- 生产构建：`npm run build`
- 生产启动：`npm run start`
- 健康检查：`/api/health`
- 文档中已有 PM2 + Nginx 的 VPS 部署说明，适合朋友内测或小范围试玩

当前仓库没有写死一个公开正式线上地址，更像“支持上线”的原型工程，而不是稳定运营中的正式产品。

## 当前玩法主干

当前 Demo 的主干已经不是早期的整月抽象行动，而是：

1. 每周开始先确定课程态度
2. 周前触发随机事件并落到具体某天
3. 逐天点击安排本周，每天 1 个行动
4. 本周全部安排后点击确认
5. 统一进行周结算
6. 月末汇总成长日志、月记、履历与结局预览

## AI 在项目中的职责边界

AI 只负责表达层，不负责规则判定。

当前约束非常明确：

- AI 可以生成月记
- AI 可以生成结局表达 / 结局报告
- AI 可以在已有结构化摘要基础上做“更像人写的”表达

AI 不应该做这些事：

- 不负责行动是否成功
- 不负责课程、学业、经济、事件、推免、考研、考公、就业等规则判定
- 不负责新增规则
- 不负责反向修正数值结果
- 不允许脱离规则层杜撰关键事实

一句话概括：规则必须代码驱动，AI 只负责把规则层已经确认的事实写得更自然。
