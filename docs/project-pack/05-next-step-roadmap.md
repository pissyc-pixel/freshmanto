# 下一阶段路线图建议

这份 roadmap 不是“还能加什么功能”的清单，而是从当前代码和风险状态出发，更适合继续推进的顺序。

## 近期最优先

### 1. 先做规则层回归，而不是继续扩功能

当前最应该优先做的是：

- 周排程到周结算链路回归
- 比赛 / 长期项目线回归
- 月末摘要、成长日志、月记、结局预览读取链路回归
- 方向提示和规则层是否一致的回归

理由：

- 项目现在不是“没系统”，而是“系统开始多了，最怕链路断”

### 2. 把当前可玩闭环打磨稳

包括：

- 周排程可解释性
- 周结算可信度
- 现金风险与生活成本的理解门槛
- 比赛入口与长期投入的真实感
- 月末页面的稳定度

### 3. 为继续试玩准备更清晰的观测材料

建议：

- 保存几局典型 run 的截图
- 整理几组典型路径样本
- 记录一次完整从大一到大三的试玩观察

## 中期要补的

### 1. 把后半程路径内容做厚

重点不是再加“抽象方向”，而是补过程内容：

- 推免资格后的现实选择
- 考研准备过程
- 公考准备与事件链
- 求职宣讲、投递、面试、offer 过程

### 2. 扩充事件池、比赛池、机会池

当前结构对了，但内容密度不够。

中期应补：

- 学院风味事件
- 城市差异事件
- 更多长期项目入口
- 更真实的实习与履历机会

### 3. 继续提升“前台可感知化”

尤其是：

- 为什么这周 / 这个月会变成这样
- 为什么某条路更近、某条路更远
- 为什么奖学金 / 推免 / 公考线会发生变化

## 后期扩展方向

### 1. 更完整的四年人生模拟

包括：

- 更细的人际关系
- 更复杂的资源流
- 情绪与心理状态长线变化
- 城市去向与毕业去向联动

### 2. 更完整的结局系统

包括：

- 更丰富的毕业状态
- 每条主分支的成功 / 普通 / 失败 / 转向细分
- 更强的“回头看一生轨迹”的总结感

### 3. 正式产品化能力

如果以后想从原型进入更长期维护，可再考虑：

- 登录与账号体系
- 观测与埋点
- 正式部署环境与运维
- 数据分析与平衡调整工具

## 一句话建议

下一阶段最合理的方向不是“继续加更多系统”，而是：

先把现有规则链路稳住，再把后半程路径内容做厚。
