# v0.2 玩法迭代说明

## 本轮目标

这一轮不是继续加大而全的新功能，而是把 v0 的基础工程闭环修成更像“大学生活模拟”的试玩体验。

核心目标：

- 从整月批处理改成按次行动、即时结算。
- 用周历表达取代平铺的 30 天时间感。
- 拆分前台日志和后台日志。
- 把月记改成更拟人的第一人称表达。
- 修正经济、压力、学习收益和翘课后果。
- 加入最小可扩展的随机事件系统。

## 阶段拆分

### 阶段 1：主 agent 规划与边界梳理

主 agent 先确认：

- 哪些模块需要改
- 哪些文件是共享边界
- 哪些地方容易冲突
- 哪些内容要坚持增量修改而不是推翻重来

本阶段产出：

- 子任务边界
- 文件所有权划分
- 联调重点

### 阶段 2：并行 subagent 实现

#### Subagent A：时间流与行动系统重构

负责：

- 按次行动与即时结算
- 4 周周历表达
- 主游戏页逐步推进体验
- 翘课后果链从“直接扣学业”改成“风险 / 花费 / 补救”

主要范围：

- `core/game-engine`
- `core/resolvers/schedule.ts`
- `core/resolvers/attendance.ts`
- `lib/demo/run-service.ts`
- `app/actions.ts`
- `app/game/page.tsx`
- `components/action-plan-form.tsx`

#### Subagent B：经济 / 平衡 / 随机事件

负责：

- 每月固定支出
- “吃大餐”
- 学习收益削弱和递减
- 压力惩罚
- 最小状态驱动随机事件

主要范围：

- `core/resolvers/actions.ts`
- `core/resolvers/events.ts`
- `core/resolvers/semester.ts`
- `data/events.ts`

#### Subagent C：前台日志与拟人月记

负责：

- 前台日志与后台日志拆分
- 月记和结局 fallback / prompt 风格改造
- 月结算、月记、履历页面的可读性

主要范围：

- `app/settlement`
- `app/journal`
- `app/resume`
- `app/ending`
- `components/log-feed.tsx`
- `lib/ai/reports.ts`
- `core/prompts`
- `lib/demo/options.ts`

### 阶段 3：主 agent 整合联调

主 agent 最终负责：

- 整合三块提交
- 修复共享类型与模块接口
- 跑通测试、lint、build
- 更新 README 和架构文档
- 补齐跨模块体验断点

## 本轮整合重点

主 agent 在整合阶段额外补了两类关键接缝：

1. “吃大餐”正式进入共享类型、server action 校验和 UI 入口。
2. “吃大餐”被修成即时消费：
   - 会立即结算
   - 不推进周历
   - 不重复触发一次课程结算

同时，主 agent 也把前台日志重新收口成规则事实到中文叙事的映射层，避免玩家直接看到 `event:*`、`roll-call-risk:*` 这类机器标记。

## 当前结果

本轮完成后，Demo 已经具备：

- 逐步行动与即时反馈
- 周历表达
- 固定生活成本与即时消费选择
- 更合理的学习收益和压力负反馈
- 最小随机事件系统
- 前后台日志分离
- 更拟人的月记 fallback 和 AI 输入边界

## 后续建议

- 继续把周推进细化为周内多个时间块。
- 扩展事件池和补救链。
- 丰富社交资源、人脉帮助和恋爱相关占位事件。
- 让结局维度覆盖就业、城市去向、关系和心理状态。
