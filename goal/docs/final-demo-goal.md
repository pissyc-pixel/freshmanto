三、硬性边界
━━━━━━━━━━━━━━━━━━━━

1. AI 不能判定规则。
2. AI 不能决定奖学金、竞赛、实习、offer、推免、考研、就业、毕业结果。
3. AI 只能基于规则层 FACTS 写月记、成长时间线润色、结局信、重要结果说明。
4. 不能编造 FACTS 里没有的奖学金、竞赛、实习、offer、录取、毕业结果。
5. 演示存档可以是精心设计的 deterministic fixture，但必须写入真实游戏状态，能继续玩，不能只是展示页面。
6. 不要使用真实企业名。
7. 不要新增大型依赖。
8. 不要做赛博风、霓虹风、抽卡风、3D、粒子、复杂背景。
9. 不要把失败结果写得嘲讽或冷冰冰。
10. 不要 force push。
11. 不要直接合并 master。
12. 完成后提交 commit 并报告，不要自行合并主分支。

{FACTS_JSON}

十五、AI 表达层总原则
━━━━━━━━━━━━━━━━━━━━

所有 AI 参与写作的内容，都必须人性化、具体、温暖、有情感回收。

AI 不能写成：
- 系统总结
- 后台报告
- 数据罗列
- 公众号鸡汤
- 班主任评语
- 空泛夸奖

AI 应该写成：
- 月记像普通学生手机备忘录
- 成长时间线像四年档案节点
- 奖学金像正式证书
- 竞赛像成果被看见
- 实习像第一次连接校园外世界
- 推免 / 考研像录取通知
- 就业 offer 像未来选择
- 最终结局像写给四年后的信

所有文本都必须基于 FACTS。
不能编造 FACTS 里没有的奖学金、竞赛、实习、offer、学校录取、朋友、老师、公司、导师、毕业结果。

━━━━━━━━━━━━━━━━━━━━
十六、重要结果 UI 情感冲击
━━━━━━━━━━━━━━━━━━━━

freshmanto 的重要结果 UI 不能只是系统通知。

必须有：

等待 → 揭晓 → 被认可 → 回看付出 → 留下纪念

覆盖：

1. 奖学金证书
2. 竞赛获奖 / 结项结果
3. 实习录取 / 实习完成
4. 推免 offer
5. 考研录取 offer
6. 就业 offer
7. 毕业结局
8. 最终结局报告

成功结果要让玩家开心、松一口气、有被认可的感觉。
普通结果要让玩家觉得努力没有白费。
失败结果要温和解释原因，并保留下一步可能性。

失败不能写成：
你没有获得奖学金。
你没有 offer。
你考研失败。

要写成：
这一次没有拿到结果，不代表这一年完全失败。
只是从评定结果看，你的学业稳定性和履历加分项，还没有形成足够明显的优势。
它更像是一个提醒：下一步还可以调整。

所有重要结果必须用：

- 证书
- 通知书
- offer letter
- 毕业档案
- 最后一封信

而不是普通 alert、toast 或后台卡片。

保留现有轻量 / 半天 / 全天行动可用性限制：

1. 夜间空闲时，只能执行轻量行动。
2. 半天空闲时，可以执行轻量和半天行动。
3. 全天空闲时，可以执行轻量、半天和全天行动。

轻量行动在半天或全天执行时，可以获得更完整时间带来的增强。

但：

1. 不要在代码里写死简单粗暴的 x2 / x3。
2. 具体数值应保持可调，并通过测试和试玩逐步平衡。
3. UI 不显示倍率，只用自然语言暗示：
   - 今晚做一点
   - 拿半天认真做
   - 用一整天推进
4. 同一目标下，高阶行动必须有不可替代价值。
5. 全天执行“放松一下”不能强过“休息一天”。
6. 全天执行“课程复习”不能完全替代专门的全天学习/补课行动。
7. 普通轻量行动不能替代竞赛、实习、考研等专门长期线行动。
8. 竞赛线不套用轻量行动时间倍率。

每周开始时，玩家要看到：

- 本周课程态度
- 本周事件预告
- 本周现金风险
- 本周压力/心情状态
- 本学期正在推进的竞赛/项目
- 当前长期方向提醒

每张行动卡必须回答：

1. 这是什么行动？
2. 它消耗什么时间？
3. 它会带来什么收益和代价？
4. 它会推进哪条长期线？
5. 它为什么解锁？

行动分类：

- 学习 / 课程
- 生活 / 休息
- 金钱 / 兼职
- 社交 / 心情
- 竞赛 / 项目
- 实习 / 就业
- 考研 / 推免 / 考公
- 事件相关

周结算要变成“这一周过完了”，不是数字列表。

结构：

1. 一句话周总结
2. 每天卡片回放
3. 本周状态变化
4. 长期项目推进
5. 下周提醒

十八、经济 / 压力 / 心情反馈
━━━━━━━━━━━━━━━━━━━━

金钱、压力、心情不是装饰数值。

金钱必须有账本：

收入：
- 月初生活费
- 奖学金
- 兼职收入
- 向家里要钱
- 实习补贴
- 事件收入

支出：
- 基础生活费
- 行动额外花费
- 社交 / 娱乐
- 交通 / 实习 / 求职成本
- 事件支出

offer 的薪资不能直接加到大学期间现金里。

金钱风险：

正常：余额足够覆盖本周基础开销
预警：现金有点紧
危险：本周可能不够花
严重：生活费快撑不住了

低余额不能只变红，要影响选择和提醒。

向家里要钱：

每月最多 1 次，或连续使用效果下降。
金额受家庭背景影响。
短期补现金，但可能带来一点压力或成就感下降。
文案不能羞辱玩家。

压力分层：

0-39：压力可控
40-59：有点紧
60-74：压力偏高
75-89：压力过高
90+：接近崩溃

压力过高影响：

- 学习效率下降
- 考研 / 竞赛 / 面试发挥打折
- 心情更容易下降
- 月记更疲惫
- 结局证据链记录

心情分层：

70+：状态不错
50-69：普通
35-49：偏低
20-34：很低
<20：非常低

心情低影响：

- 学习 / 竞赛 / 考研效率下降
- 社交行动收益降低
- 压力恢复变慢
- 月记更消沉

必须有补救路径：

- 休息
- 社交
- 吃顿好的
- 兼职
- 向家里要钱
- 减少高压行动

关键状态变化要通过弹窗或结果卡反馈。

十九、随机事件 / 学期节奏
━━━━━━━━━━━━━━━━━━━━

事件要有四层：

1. 当月月历提前展示本月值得注意的日子。
2. 周开始弹窗告诉玩家这周发生什么。
3. 当天安排时影响行动池 / 时间结构。
4. 周结算或月结算进入回顾、月记、时间线。

事件类型：

A. 强制占用类
- 班会
- 导员通知
- 材料补交
- 签到严查
- 期末考试周
- 毕业材料确认

必须区分 soft / hard。

soft event：
只占用一点时间，不应该完全锁死整天行动。

hard event：
真的占用大半天或全天，必须说明。

B. 信息机会类
- 竞赛宣讲
- 企业宣讲
- 考研讲座
- 推免说明会
- 公考经验分享

C. 履历机会类
- 大创招募
- 竞赛报名
- 老师项目
- 实习内推
- 校内实践

D. 经济事件类
- 生活费晚到账
- 临时材料费
- 兼职机会
- 家里额外支持

E. 情绪 / 压力事件类
- 连续赶 ddl
- 期末焦虑
- 被老师点名
- 一次很累的面试
- 一次让人开心的聚会

F. 关键节点类
- 竞赛评奖
- 奖学金评定
- 考研开启
- 推免申请
- 考研结果
- offer 选择
- 最终结局

经济事件、情绪/压力事件必须在每周开始以弹窗形式出现，并告诉玩家变动了什么：

模板：

【事件标题】

发生了什么：
一两句话自然描述。

本周变化：
- 金钱下降 / 上升
- 压力上升 / 下降
- 心情上升 / 下降
- 某行动开放 / 某天时间受限

这周提醒：
一句可操作建议。

当月月历要提前展示事件层级：

普通事件：轻量提示
风险事件：预警样式
机会事件：星标/机会样式
正式结果：证书/通知样式
人生节点：高亮关键节点

可以提前展示：
“第 4 周：竞赛评奖”
“第 13 月：奖学金评定”

不能提前泄露：
“你会获得二等奖”
“你会拿国家奖学金”

二十、履历档案 / 证据链
━━━━━━━━━━━━━━━━━━━━

履历页不是字段列表，而是成长证据板。

必须回答：

1. 我现在有什么可以写进履历？
2. 这些东西支撑哪条未来路线？
3. 我还缺什么？

履历页模块：

1. 学业档案
2. 奖学金记录
3. 竞赛 / 项目经历
4. 实习 / 实践经历
5. Offer / 录取记录
6. 路线证据链

奖学金记录格式：

第 1 学年｜市级奖学金｜6000 元
第 2 学年｜国家奖学金｜10000 元

竞赛经历格式：

第 1 学年｜电子设计竞赛｜校级二等奖
第 2 学年｜大创项目｜完成项目

实习经历格式：

第 2 学年｜普通实习｜本地企业运营助理
第 3 学年｜较好实习｜二线城市数据分析实习

Offer / 录取记录：

推免 Offer｜天津大学｜已接受
考研录取｜211 高校｜已拒绝
就业 Offer｜一线城市产品运营岗｜待选择

路线证据链：

推免方向：
支撑：
- GPA 稳定
- 获得奖学金
- 有竞赛经历

短板：
- 高等级竞赛还不够
- 科研经历还不够强

就业方向：
支撑：
- 有实习
- 有项目
- 社交和信息获取较好

短板：
- 高质量实习还不够
- offer 竞争力仍需提升

空状态要温和：

这部分还没有形成记录。
大学前期本来就是慢慢积累的阶段。
后续参加竞赛、项目、实习或获得奖学金后，这里会一点点变厚。

不能编造不存在的履历。

二十一、UI / 动效总规范
━━━━━━━━━━━━━━━━━━━━

视觉定位：

浅绿色、奶白色、柔和卡片、大学生活档案本、成长记录册、轻模拟游戏 UI。

不是后台系统。
不是赛博游戏。
不是 landing page。
不是二次元手游。
不是企业 dashboard。

关键词：

- 温和
- 真实
- 有生活感
- 有成长档案感
- 清晰
- 稳定
- 轻微游戏感

全局动效必须短、轻、明确。

允许：

- fade
- translateY 轻微位移
- scale 0.98 到 1
- 轻微 shadow transition
- 轻微 progress bar transition
- 列表项 stagger
- loading spinner / shimmer

禁止：

- 大幅弹跳
- 粒子
- 彩带
- 翻牌抽卡
- 复杂 3D
- 霓虹光效
- 大面积背景流动
- 页面元素一直晃动

时间建议：

hover：120-160ms
button press：80-120ms
modal enter：160-220ms
card enter：160-220ms
page section enter：180-240ms
list stagger：每项 35-50ms
ending reveal：整体不超过 2 秒

必须支持 prefers-reduced-motion。

重要结果 UI 要像：

- 奖学金证书
- 竞赛成果通知
- 实习机会通知
- 推免 offer letter
- 考研录取通知书
- 就业 offer letter
- 毕业档案
- 最后一封信

普通结果不能冷冰冰。
